## Friendly status and error copy

**Loading:** `Opening page 17...`

**Image error:** `This page did not load. Try again, or move to the next page.`

**First-page boundary:** `You are at the first page.`

**Last-page boundary:** `That is the last page in this issue.`

The modal should close with its visible Close button or the Escape key. When it closes, keyboard focus should return to the thumbnail that opened it.

# SEO and publishing checklist

## Homepage

- Use one H1 only: `Three issues. 163 pages worth exploring.`
- Give every issue card a descriptive H2 title.
- Write a unique editorial summary for every issue.
- Link issue cards to permanent URLs, not JavaScript-only actions.
- Use a self-referencing canonical URL.
- Add Organization or WebSite structured data where accurate.
- Use a three-cover social image or a representative current cover.

## Issue pages

- Give each issue a unique SEO title, meta description, H1, and 50-100 word introduction.
- Include the issue number, editorial title, publication date, page count, topics, and contributors where known.
- Add CreativeWork or PublicationIssue structured data only when the supplied properties are accurate.
- Link back to the homepage and across to adjacent issues.
- Include each canonical issue URL in the XML sitemap.

## Images

- Use real HTML image elements, not CSS background images, for covers and magazine pages.
- Use descriptive filenames such as `concrete-curve-issue-01-page-017-thumb-153x108.webp`.
- Set intrinsic dimensions on thumbnails: `width="153" height="108"`.
- Supply AVIF or WebP with a JPEG fallback where practical.
- Lazy-load gallery images below the initial viewport, but do not lazy-load the likely hero/LCP image.
- Use an image sitemap if image discovery is an important acquisition channel.
- Keep each image URL stable wherever it appears.

## Copy quality

- Write for people who are curious, not for algorithms.
- Prefer names, subjects, places, and real editorial details over abstract adjectives.
- Keep calls to action conversational and specific: `Open Issue 02`, not `Learn more`.
- Do not label every issue “inspiring,” “innovative,” or “unique.” Show what makes it different.
- Update the homepage whenever a new issue becomes the latest issue.
